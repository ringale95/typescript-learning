What is TypeScript? How is it different from JavaScript?

ECMAScript(ES6/2015) is a standard which is implemented by Javascript. It introduced arrow functions, modules(import and export), Template literals, Destructuring in Javascript

TypeScript is JavaScript with type checking. It is a programming language created to address the shortcomings of JavaScript. It can be considered as a superset or a wrapper of JavaScript.

TypeScript compiler uses an older version of JavaScript: ES5 (ECMAScript 5)

Benefits of TypeScript:
  • Static Typing
    • There are two types of typing: Statically Typed and Dynamically Typed
    • Statically Typed: Examples include C#, Java, C++. These languages provide compile-time errors for variables.
    • Dynamically Typed: Examples include JavaScript, Python, Ruby. The type of a variable is determined at runtime, which can be problematic.
      • let x: number = 10;         
z = "a";                      // gives compile-time error
  • Code Completion
  • Refactoring -> Writing more clean and modular and meaningful code
  • Shorthand Notations
Drawbacks of TypeScript:
  • Compilation: Browsers do not understand TypeScript, so the code must be converted to JavaScript using the TypeScript compiler. This process is called Transpilation.
  • Discipline in writing code

TypeScript Configuration:

To generate a configuration file for TypeScript, use the command: tsc --init

This generates a tsconfig.json file.

  • target: Specifies which JavaScript version is used by TypeScript. This can be manually set to any ES version.
  • rootDir: Specifies the directory that contains the source files. It should be ./src.
  • outDir: Specifies the directory for the JavaScript files generated by the TypeScript compiler. It should be ./dist.
  • removeComments: Removes comments from the TypeScript code, making the JavaScript code shorter.
  • noEmitOnError: Prevents the TypeScript compiler from generating JavaScript files if there are errors in the code. Uncomment this option to enable it.
  • noUnusedLocals: Enable error reporting when local variables aren't read
  • noUnusedParameters: Raise an error when a function parameter isn't read
  • noImplicitReturns:Enable error reporting for codepaths that do not explicitly return in a function
  • noImplicitAny: Compiler will warn you about variables that are inferred with any type.

// can also write
let sales: 123_456_789;

Functions:

    console.log(document); 
}
    
function render(document : any){           // we explicitly annotate it
    console.log(document); 
}

Objects:

In JS objects are dynamic so that their shape can change like for example below:

let employee = { id : 1}
employee.name = 'Mosh';

In TS objects are strictly defined properly:

let employee : {
    id: number;
    name?: string                   //optional so no need to supply while initializing
} = {id: 1};


let employee : {
    id: number;
    name: string                   //optional so no need to supply while initializing
} = {id: 1, name: 'Mosh}';

employee.id = 0                     // we can change the value of id which is not good so we define readonly


let employee : {
    readonly id: number;
    name: string                   //optional so no need to supply while initializing
} = {id: 1, name: 'Mosh}';
//Define function in objects 
let employee : {
    readonly id : number,
    name : string,
    retire : (date:Date) => void
    } = {
    id : 1,
    name : 'Raveena',
    retire: (date: Date) => {
    console.log(date);
    }
}
